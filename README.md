# 🏎️ Line Tracer: 실시간 라인 센서 신호 처리 및 고속 조향 제어 시스템

## 1. 프로젝트 개요 (Introduction)
본 프로젝트는 **임베디드 시스템 설계 과목 팀 프로젝트**로 진행한 자율주행 라인트레이서 개발 프로젝트입니다.
제한된 마이크로컨트롤러(MCU) 하드웨어 자원 내에서 시스템의 **실시간성(Real-time)**을 확보하기 위해, Timer 인터럽트 기반의 **DDA(Digital Differential Analyzer)** 알고리즘을 직접 구현하여 스텝 모터를 제어한 것이 본 프로젝트의 핵심입니다. 

### 🛠️ 개발 환경 및 하드웨어 사양
* **Language / IDE:** C/C++ (Atmel Studio)
* **Controller:** 아두이노 ATmega 2560
* **Sensor:** TSL1401 Line Scan Camera (128 pixels)
* **Actuator:** Step Motor (via DDA Control)
* **Control Frequency:** 200Hz Main Loop / 20kHz Timer Interrupt

---

## 2. System Architecture & Control Logic

### 2.1 DDA 알고리즘 기반 STEP 펄스 생성 (Low-level Control)
아두이노의 기본 라이브러리에 의존하지 않고, 레지스터 제어를 통한 하드웨어 타이머 인터럽트를 직접 구현했습니다.
* `Timer1 ISR(20kHz)` 내에서 DDA 알고리즘을 수행하여 정밀한 스텝 펄스를 생성.
* 메인 루프(200Hz)에서 계산된 좌/우 목표 속도(`b_left`, `b_right`)를 바탕으로, 실시간으로 펄스 주기를 가변하여 모터의 선형적인 가감속(Acceleration/Deceleration) 제어를 달성했습니다.

### 2.2 환경 적응형 센서 신호 처리 (Robust Line Detection)
조명 변화 및 빛 반사에 취약한 1D 라인 스캔 센서의 하드웨어적 한계를 소프트웨어 알고리즘으로 극복했습니다.
* **동적 임계값(Dynamic Thresholding):** 고정된 Threshold의 한계를 극복하기 위해, 매 프레임의 유효 픽셀 구간 내 최댓값과 최솟값의 평균을 계산하여 Threshold를 실시간으로 갱신했습니다.
* **단일 차선 보정 및 탈선 복구:** 한쪽 차선을 놓칠 경우 조향 중심(`Center_ind`)을 1씩 완만하게 보정하여 급조향을 방지했으며, 명암 대비차가 150 미만으로 떨어지면 '탈선'으로 규정하고 저속 후진을 통해 라인을 재탐색하는 복구 로직을 설계했습니다.
* **정지선 쿨다운(Cooldown) 로직:** 정지선의 중복 인식을 방지하기 위해 `1,000 프레임` 동안 재검출을 무시하는 상태 머신(State Machine) 로직을 도입했습니다.

### 2.3 오차 변화량 기반 조향 및 가속 최적화 (PD Control Logic)
주행 랩 타임(Lap Time) 단축과 코너링 안정성이라는 두 마리 토끼를 잡기 위해 **가변 이득 및 모션 프로파일**을 적용했습니다.
* **에러 기반 가변 이득(Variable Gain):** 중심 오차(`abs_err`)가 8.0 미만인 직선 구간은 부드러운 조향(`K=15.0`), 코너 구간에서는 강력한 조향(`K=80.0`)을 인가했습니다.
* **오차 변화량(derr) 감지:** 오차(`err`)뿐만 아니라 오차의 변화량(`derr`)을 동시에 분석하여 코너 진입 시점을 민감하게 포착, 선제적으로 감속(`-2000`)하고 직선에서는 최대 속도(`top_speed`)로 가속하는 정밀 주행을 구현했습니다.

## 3. 트러블슈팅 및 고찰 (Troubleshooting & Analysis)

### 💡 고속 주행 시 코너 이탈 문제 (PD 제어 도입)
* **Issue:** 현재의 오차(`err`) 값만으로 비례 제어를 수행할 경우, 고속 진입 시 감속 시점이 늦어져 관성에 의해 차량이 트랙을 이탈하는 현상이 발생했습니다.
* **Solution:** 차량이 궤적에서 벗어나는 '속도' 자체를 제어 변수로 사용하기 위해 오차 변화량(`derr`, 미분항 역할)을 제어 조건에 추가했습니다. 이를 통해 코너 진입을 선제적으로 감지하고 민감하게 반응하는 조향 안정성을 확보했습니다.

### 💡 센서 포화(Saturation) 현상의 하드웨어적 해결
* **Issue:** 강한 외부 햇빛으로 인해 광량이 과다해지면, 검은색 차선마저 픽셀 최댓값으로 포화(Saturation)되어 라인 인식이 완전히 불가능해지는 문제가 발생했습니다.
* **Solution:** 소프트웨어적인 필터링만으로는 한계가 있다고 판단, **라인 스캔 센서 구동 회로의 저항값을 키워 물리적인 Bias를 낮추는 하드웨어 튜닝**을 단행했습니다. 이를 통해 SW/HW 간의 상호 보완적 시스템 설계의 중요성을 체득했습니다.
